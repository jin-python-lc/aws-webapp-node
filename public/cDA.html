試験1 - 分野別

データ・アナリティクス

＜第21問＞
あなたの会社のデータサイエンティストは、リアルタイムのストリーミングデータを処理・分析できるサービスを探しています。ストリーミングデータの問い合わせには、標準的なSQLクエリを使用したいと考えています。これらの要件を満たすのは、どのAWSサービスの組み合わせでしょうか。

答え：2

1. ElastiCache + EMR

2. Kinesis Data Stream + Kinesis Data Analytics

3. DynamoDB + EMR

4. Kinesis Data Stream + Kinesis Data Firehose


・1-ElastiCacheはインメモリデータベースのキャッシュサービスであり、ストリーミングデータには使用されないため、ElastiCache + EMRは誤りです。EMR（Elastic Map Reduce）はホスト型のHadoopフレームワークであり、ストリーミングデータの分析には使用されません。

・2-Amazon Kinesis Data Analyticsは、リアルタイムのストリーミングデータを処理・分析する最も簡単な方法です。Kinesis Data Analyticsは、標準的なSQLクエリを使用してKinesisデータストリームを処理し、Kinesis StreamsやKinesis Firehoseからデータを取り込むことができます。

・3-DynamoDB + EMRは誤りです。DynamoDBはNoSQLデータベースで、ストリームからのデータの保存には使用できますが、データの処理や分析、SQLクエリでの問い合わせには使用できません。Elastic Map Reduce（EMR）はホスト型のHadoopフレームワークで、ストリーミングデータの分析には使用できません。

・4-Kinesis Data Streams + Kinesis Firehoseは誤りです。FirehoseはSQLクエリの実行には使用できません。

＜第24問＞
新しいアプリケーションのWebサイトには、毎秒約5万件のリクエストがあり、同社は複数のアプリケーションを使用して、Webサイト上のユーザーのナビゲーションパターンを分析し、ユーザーエクスペリエンスをパーソナライズできるようにしたいと考えています。Webサイトのページクリック数を収集し、ユーザーごとに順次処理するために、次のうちどのサービスを選択すべきですか。

答え：2

1. SQS 標準キュー

2. Kinesis Data Stream

3. SQS FIFOキュー

4. CloudTrail


・1-標準のSQSキューは、メッセージが順番に処理されることを保証せず、FIFOSQSキューは1秒間に必要なトランザクション数にスケーリングしません。

・2-Amazon Kinesisストリームでは、1秒あたり最大1 MiBのデータ、またはシャードあたりの書き込みで1秒あたり1,000レコードが許可されます。シャードの数に制限はないため、KinesisStreamsを簡単にスケーリングして1秒あたり50,000を受け入れることができます。

・3-SQSはデータのストリーミングに最適ではありません。この問題の場合にはKinesisがより優れたソリューションとなります。

・4-CloudTrailは監査に使用されるため、ここでは役に立ちません。

＜第26問＞
外部のWebサイトで動作するアプリケーションが、Amazon API GatewayへのAPIコールを使用して、貴社のWebサイトへのリクエストを開始しようとしています。その際、次のようなエラーが発生してリクエストが失敗するという問題が報告されています。"Cross-Origin Request Blocked: The Same Origin Policy disallows reading the remote resource"あなたは問題を解決するように求められています。最も可能性の高い解決策を選択してください。

答え：3

1. APIのACLを変更する。

2. IAMポリシーがAPIアクセスを許可していないため変更する。

3. API gatewayで選択したメソッドを使用して、APIリソースでCORSを有効にする。

4. リクエストがSSL/TLSで保護されていないため、変更する。


・1-不正解です。

・2-不正解です。

・3-CORSをサポートするには、REST APIリソースは、少なくともFetch標準で義務付けられている次の応答ヘッダーを使用してOPTIONSプリフライト要求に応答できるOPTIONSメソッドを実装する必要があります。Access-Control-Allow-Methods、Access-Control-Allow-Headers、Access-Control-Allow-Origin。

・4-不正解です。

＜第35問＞
ある調査会社は、Amazon S3でデータレイクソリューションを運用し、巨大なデータセットを分析しています。このソリューションにおいて、SQLクエリは低頻度で行われます。同社はインフラストラクチャのコストを最小限に抑えたいと考えています。これらの要件を満たすサービスを選択してください。

答え：1

1. Athena

2. Aurora

3. Redshift Spectrum

4. RDS for MySQL


・1-Amazon Athenaは、標準SQLを使用してAmazonS3のデータを簡単に分析できるインタラクティブなクエリサービスです。Athenaはサーバーレスであるため、管理するインフラストラクチャはなく、実行するクエリに対してのみ料金を支払います。これにより、頻度の低いクエリのインフラストラクチャコストを最小限に抑えるという要件が満たされます。

・2-Amazon RDSとAuroraは、S3でデータセットを分析するのに適したソリューションではありません。

・3-Amazon RedShift Spectrumは、Amazon Redshiftの機能であり、ロードやETLを必要とせずに、AmazonS3の非構造化データのエクサバイトに対してクエリを実行できます。ただし、RedShiftノードはEC2インスタンスで実行されるため、クエリの頻度が低い場合、インフラストラクチャのコストを最小限に抑えることはできません。

・4-Amazon RDSとAuroraは、S3でデータセットを分析するのに適したソリューションではありません。

＜第65問＞
ビッグデータ分析企業は、データをS3ベースのデータレイクからRedshiftクラスターに移行および変換する毎日のETLプロセスを最適化したいと考えています。サーバーレス環境でこの日常業務を管理するのに最適なAWSサービスを選択してください。

答え：3

1. DMS

2. Data Pipeline

3. Glue

4. EMR


・1-DMSは、データベースをAWSに簡単かつ安全に移行するのに役立ちます。オンプレミスからAWSへのデータベースの移行、またはオンプレミスのソースとAWSのソース間のデータベースレプリケーションが必要なユースケースに適します。設問ではAWSサービス間のデータ移行と変換が要件となっているため不適切です。

・2-このオプションは基盤となるEC2インスタンスを必要とするため、サーバーレスソリューションではありません。

・3-Glueは、サーバーレスApacheSpark環境で実行されるマネージドETLサービスを提供します。これにより、基盤となるコンピューティングリソースの構成と管理について心配することなく、ETLジョブに集中できます。

・4-EMRは、膨大な量のデータを簡単かつコスト効率よく処理するためのWebサービスです。基盤となるEC2インスタンスを必要とするため、サーバーレスソリューションではありません。

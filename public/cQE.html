試験1 - 分野別

キューイング

＜第7問＞
社内用アプリケーションを運用しています。フロントエンド処理とバックエンド処理の2つのバッチモジュールで構成され、フロントの処理後、バックエンド処理を行っています。現在、このWebアプリケーションのユーザー数が増加しバックエンド処理が遅くなっています。そこでAmazon ECSにてマイクロサービス化し、ユーザー増加に対応できるアーキテクチャを検討しています。適切なソリューションを選択してください。

答え：1

1. フロントエンド処理とバックエンド処理の連携にSQSを使用する。

2. フロントエンド処理とバックエンド処理の連携にSNSを使用する。

3. フロントエンド処理とバックエンド処理の連携にS3を使用し、S3バケットへのデータの登録をトリガーとする。

4. フロントエンド処理とバックエンド処理の連携にEFSを使用し、データの共有を行う。


・1-SQSによってフロントエンドからの要求をキューイングし、バックエンド処理と連携、処理の並列化をすることができます。

・2-SNSはメッセージ通知サービスです。

・3-S3は、大容量のデータを格納できる耐久性の高いオブジェクトストレージサービスです。

・4-EFSは、EC2インスタンス間でファイル共有するサービスです。

＜第25問＞
Amazon EC2上で実行されているアプリケーションは、データ処理を実行するためにLambda関数を非同期的に呼び出す必要があります。サービスはデカップリングする必要があります。どのサービスがコンピュートサービスをデカップリング（分離）するために使用できるでしょうか。

答え：2

1. AWS Step Functions

2. Amazon SNS

3. AWS Config

4. Amazon MQ


・1-AWS Step Functionsは、ワークフローサービスです。この設問に最適なソリューションではありません。

・2-Amazon SNSの通知を処理するためにLambda関数を使用することができます。Amazon SNSは、トピックに送信されるメッセージのターゲットとしてLambda関数をサポートしています。このソリューションは、Amazon EC2アプリケーションをLambdaから切り離し、Lambda関数が呼び出されることを保証します。

・3-AWS Configは継続的なコンプライアンスに使用されるサービスであり、アプリケーションデカップリングはできません。

・4-Amazon MQはSQSに似ていますが、AWSに移行される既存のアプリケーションに使用されます。SQSは、クラウドで作成される新しいアプリケーションに対して使用する必要があります。

＜第37問＞
次のうち、SQSの処理の重複を防ぐために利用できるものを選択してください。

答え：2

1. ロングポーリング

2. 可視性タイムアウト

3. MQ

4. バッファリング


・1-ロングポーリングでは、キューが空の場合にメッセージを取得できるまで1〜20秒の設定された時間待機されることができます。

・2-可視性タイムアウトは、メッセージを取得した際に、他の受信者に一定時間(デフォルト30秒)見せないようにすることで、処理の重複を防ぎ、リクエストの数を減らすことができます。

・3-MQはメッセージキューイングサービスです。

・4-重複を防ぐとあるため、バッファリングは関係ありません。

＜第52問＞
1つ以上のコンポーネントに障害が発生した場合に、ソフトウェアコンポーネント間のメッセージが失われないようにするために使用されるサービスを選択してください。

答え：2

1. AWS Direct Connect

2. SQS

3. AmazonConnect

4. SES


・1-Direct Connectは、オンプレミスとAWSの間に専用のネットワーク接続を確立するために使用されるクラウドサービスソリューションです。

・2-SQSは、メッセージを失ったり、他のサービスを利用できるようにしたりすることなく、任意のボリュームのソフトウェアコンポーネント間でメッセージを送信、保存、および受信できるようにするフルマネージドメッセージキューサービスです。

・3-Amazon Connectは、企業が低コストでカスタマーサービスを簡単に提供できるようにするクラウドベースのコンタクトセンターサービスです。

・4-Amazon SESは、企業や開発者向けの柔軟で手頃な価格の拡張性の高いEメールメッセージングプラットフォームです。

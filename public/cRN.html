＜第27問＞
ウェブサイトは、AmazonCloudFrontディストリビューションのオリジンとして機能するApplicationLoad Balancer（ALB）の背後にあるAutoScalingグループのAmazonEC2インスタンスで実行されます。SQLインジェクション攻撃から保護するために、AWS WAFを使用しています。セキュリティログを確認したところ、Webサイトへのアクセスをブロックする必要がある外部の悪意のあるIPが明らかになりました。アプリケーションを保護するために必要な設定を選択してください。

答え：2

1. ALBの背後にあるターゲットグループのEC2インスタンスのネットワークACLを変更して、悪意のあるIPアドレスを拒否する。

2. WAFの設定を変更して、悪意のあるIPアドレスをブロックするIP一致条件を追加する。

3. ALBの背後にあるターゲットグループのEC2インスタンスのセキュリティグループを変更して、悪意のあるIPアドレスを拒否する。

4. CloudFrontディストリビューションのネットワークACLを変更して、悪意のあるIPアドレスの拒否ルールを追加する。


・1-EC2インスタンスのサブネット内のデータ送信元IPアドレスは、ELBのIPアドレスとなるため誤りです。

・2-AWSWAFクラシックでは「IP一致条件」を作成し、2019年バージョンのAWS WAFでは「IPセット一致ステートメント」を作成します。IP一致条件/ IPセット一致ステートメントは、一連のIPアドレスとアドレス範囲に対してWeb要求の発信元のIPアドレスを検査します。これを使用して、要求の発信元のIPアドレスに基づいてWeb要求を許可またはブロックします。

・3-セキュリティーグループで特定のIPアドレスを拒否することはできません。

・4-CloudFrontはサブネットの中に無いため、ネットワークACLは適用されません。

＜第38問＞
あなたの会社では、AWS上でアプリケーションを運用しています。あなたはAWSサービスごとにコストを可視化して分析する環境構築を任されました。利用すべきサービスを選択してください。

答え：3

1. Consolidated Billing

2. Athena

3. Cost Explorer

4. Budgets


・1-Consolidated Billingは、AWSの契約アカウントをまとめて１つの支払いにすることができるサービスです。

・2-Amazon Athenaは、標準的なSQLを使用してAmazon S3内のデータを直接分析することを容易にするインタラクティブなクエリサービスです。

・3-Cost Explorerは、コストと使用状況の詳細を表示および分析するために使用できるツールです。

・4-AWS Budgetsは、予算を設定して、コストまたは使用量が予算額や予算量を超えたときにアラートを発信できるサービスです。

＜第47問＞
ある企業は、AWS GlueとLambdaを使用してS3ベースのデータレイクでデータを処理するワークフローを管理するためのオーケストレーションソリューションを探しています。ソリューションアーキテクトとして、次のAWSサービスのうち、このケースにおいて開発努力が最も少なくなるものを選択してください。

答え：4

1. Batch

2. EMR

3. Simple Workflow Service (SWF)

4. Step Function


・1-Batchは、AWSで数十万のバッチコンピューティングジョブを簡単かつ効率的に実行できるようにするバッチ管理機能のセットです。Batchを使用してワークフローを調整することはできないため、これは正しくないオプションです。

・2-EMRは、Apache Spark、Apache Hive、Apache HBase、Apache Flink、Apache Hudi、Prestoなどのオープンソースツールを使用して大量のデータを処理するための業界をリードするクラウドビッグデータプラットフォームです。EMRを使用してワークフローを調整することはできないため誤りです。

・3-SWFは、開発者が並列または順次の行程を持つバックグラウンドジョブを構築、実行、およびスケーリングするのに役立ちます。SWFを使用すると、オーケストレーションロジックを完全に制御できますが、アプリケーションの開発が複雑になります。したがって誤りです。

・4-AWS Step Functionsを使用すると、LambdaやGlueなどの複数のAWSサービスをサーバーレスワークフローに調整および調整できます。ステップ関数は、Glue ETLジョブとラムダ関数が順番に実行され、指定されたユースケースで定義されたワークフローに従って正常に完了することを保証できます。したがって、ステップ関数が最適なソリューションです。

＜第50問＞
次のサービスのうち、AWSサービス利用者がAWSとの契約を管理できるものを選択してください。

答え：4

1. AWS Certificate Manager

2. AWS Systems Manager

3. AWS Organizations

4. AWS Artifact


・1-AWS Certificate Managerは、AWSサービスおよび内部接続リソースで使用するためのパブリックおよびプライベートのSSL/TLS証明書を簡単にプロビジョニング、管理、およびデプロイできるサービスです。

・2-AWS Systems Managerは、AWSのインフラストラクチャの可視性と制御を提供します。

・3-AWS Organizationsは、複数のAWSアカウントにわたって一元的なガバナンスと管理を提供します。

・4-AWS Artifactは、AWSのコンプライアンスドキュメントとAWS契約へのオンデマンドアクセスを顧客に提供するセルフサービスの監査アーティファクト検索ポータルです。AWS Artifact Agreementsを使用して、Business Associate Addendum（BAA）などのAWS契約のステータスを確認、承認、追跡できます。

＜第54問＞
脆弱性をチェックするためにEC2インスタンスのネットワーク評価を自動で実行するAWSサービスを選択してください。

答え：3

1. ACL

2. Kinesis

3. Inspector

4. セキュリティーグループ


・1-ACLを使用することで、EC2インスタンスのネットワークアクセスの可能性をサブネットレベルで確認できますが、これは自動的には行われません。

・2-Amazon Kinesisを使用すると、ビデオおよびデータストリームをリアルタイムで収集、処理、分析できます。

・3-Amazon Inspectorは、EC2インスタンスのネットワークアクセシビリティと、インスタンスで実行されているアプリケーションのセキュリティ状態をテストするのに役立つ自動セキュリティ評価サービスです。Amazon Inspectorを使用すると、評価テンプレートを作成して、開発およびデプロイのパイプライン全体または静的な本番システムのセキュリティ脆弱性評価を自動化できます。

・4-セキュリティグループを使用して、EC2インスタンスのネットワークアクセス可能性をインスタンスレベルで確認できますが、これは自動的には行われません。

＜第55問＞
次のサービスのうち、企業がAWSのコンプライアンスを確保するのに役立つのはどれですか？

答え：4

1. CloudFront

2. CloudEndure Migration

3. CloudWatch

4. CloudTrail


・1-Amazon CloudFrontは、コンテンツ配信ネットワーク（CDN）サービスです。

・2-Migrationは、物理、仮想、およびクラウドベースのインフラストラクチャからアプリケーションを移行するプロセスを簡素化し、互換性の問題なしにAWSリージョンで完全に機能することを保証します。

・3-Amazon CloudWatchは、AWSリソースの使用率を監視するために使用されます。

・4-AWS CloudTrailは、AWSアカウントで実行されたすべてのアクションをログに記録するように設計されています。

＜第57問＞
AWS Well-ArchitectedFrameworkのオペレーショナルエクセレンスの柱を最もよく表しているものを選択してください。

答え：2

1. 要件を満たすためのコンピューティングリソースの効率的な使用。

2. システムを監視し、サポートするプロセスと手順を改善する機能。

3. データセンターの運用をより効率的に管理する機能。

4. 障害から適切に回復するシステムの機能。


・1-信頼性の柱についての説明のため誤りです。

・2-オペレーショナルエクセレンスの柱についての説明に適しています。

・3-パフォーマンス効率の柱についての説明のため誤りです。

・4-誤りです。

＜第59問＞
あなたは、複数のAWSコンポーネントとサービスで構成されるアプリケーションを運用しています。このアプリケーションを構成する全てのコンポーネントとサービスにおけるリクエストやレイテンシを確認・トレースできるようにしたいと考えています。適切なサービスを選択してください。

答え：2

1. CloudWatch

2. X-Ray

3. CloudWatch Logs

4. CloudWatch Events


・1-CloudWatchは、AWS上で稼働する様々なシステムやリソースのログを収集・監視・可視化するサービスです。設問ではサービスの監視要件も含まれるため誤りです。

・2-X-Rayを利用することでアプリケーションを構成する全てのコンポーネントとサービスにおけるリクエストやレイテンシを確認・トレースできます。

・3-CloudWatch Logsは、CloudTrailやVPCフローログなどの様々なAWSのログを統合的に収集するサービスです。個々のログ確認しかできないため、トレースは不可能です。レイテンシも確認できません。

・4-CloudWatch Evantsは、APIイベントでトリガーするルールを作成し、何らかのアクションを行うためのサービスです。

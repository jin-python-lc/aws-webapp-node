試験1 - 分野別

ストレージ

＜第18問＞
AWSを上でWEBアプリケーションを運用しています。このアプリケーションではユーザーの行動データをS3に蓄積しています。一部の行動データは予測不能なアクセスパターンを持っていることから、ストレージクラスが最適ではないことが判明しました。これを改善するのに最適なS3ストレージクラスを選択してください。

答え：4

1. S3 Standard

2. S3 Standard IA

3. S3 One-Zone-IA

4. S3 Intelligent Tiering


・1-Standardは、アクセス頻度の高いデータ向けです。

・2-Standard-IAは、長期保存、バックアップ、災害対策ファイルのデータストアに向きます。存続期間が長くあまり頻繁にアクセスされないデータ向けです。

・3- One Zone-IAは、アクセス頻度は低いが、必要に応じてすぐに取り出すことが必要なデータに適しています。One Zone-IAはひとつのAZにデータを保存するため、Standard IAよりもコストを20%削減可能です。

・4-Intelligent Tieringは、アクセスパターンが変化したときに 4 つのアクセス階層の間でオブジェクトを移動することで、自動的にコストを削減する唯一のクラウドストレージクラスです。高頻度アクセス用と低頻度アクセス用で階層を区別しています。

＜第20問＞
あなたはApplication Load Balancerに登録されたEC2インスタンス上でWebアプリケーションを運用しています。DNSはRoute53を利用しています。メインのWebサイトに障害が起きた際、ユーザーがアクセス可能なバックアップサイトを設置したいと考えています。次のうち適切な構成を選択してください。

答え：1

1. バックアップサイトにはS3の静的ホスティングを使用し、Route53のフェイルオーバールーティングポリシーを適用する。

2. バックアップサイトにはS3の静的ホスティングを使用し、Route53のレイテンシールーティングポリシーを適用する。

3. 別リージョンに設置した同一アプリケーションに対し、ELBのフェイルオーバールーティングを適用する。

4. メインサーバーからのリダイレクト先として、別リージョンに設置した同一アプリケーションを設定する。


・1-フェイルオーバールーティングを設定することで、メインサーバーの障害を検知した時点でS3の静的サイトへルーティングするようにできます。

・2-レイテンシールーティングはレイテンシーに基づいてトラフィックをルーティングするため、メインサーバーに障害が発生していなくてもS3へルーティングする可能性があります。

・3-ELBはリージョンを跨いだ冗長化に対応しません。

・4-メインサーバーに障害があった際にはメインサーバーが機能しないためリダイレクト処理を行うことはできません。

＜第31問＞
ある会社が、非常に重要なデータをAmazonS3バケットにアップロードしました。管理者はデータの可用性を懸念しており、データを誤って削除しないように保護するための措置を講じる必要があります。データは引き続きアクセス可能な状態である必要があり、ユーザーはデータを削除できる必要があります。データの誤削除を防ぐ方法として適切なものを選択してください。

答え：2

1. S3バケットでデフォルト暗号化を有効にする。

2. S3バケットでバージョニングとMFA削除を有効にする。

3. S3バケット内のオブジェクトのライフサイクルポリシーを作成する。

4. S3バケットにバケットポリシーを適用する。


・1-正しくありません。

・2-バージョン管理が有効なバケットからオブジェクトを削除する前に、追加の認証を追加するため正解です。

・3-この設問において、バケットポリシーは必要ありません。

・4-ライフサイクルポリシーはデータを別のストレージクラスに移動しますが、削除を保護しません。

＜第32問＞
プライベートS3バケットに保存されている複数のビデオファイルを一時的に顧客に共有する必要があります。顧客はAWSアカウントを持っていないため、許可された顧客のみがファイルにアクセスできるようにする必要があります。このアクセスを可能にする方法を選択してください。

答え：3

1. S3バケットのパブリック読み取りアクセスを有効化する。

2. 顧客のIPアドレスに対しセキュリティーグループで許可ルールを作成する。

3. 署名付きURLを発行し、顧客に配布する。

4. CloudFrontを使用し、認証ハッシュタグを用いてファイルを配布する。


・1-パブリック読み取りアクセスを有効にしても、コンテンツの配布を許可された顧客に制限することはできません。

・2-セキュリティグループはS3バケットには適用されません。

・3-事前に署名付きURLを発行し、オブジェクトをダウンロードするための期限付きのアクセス許可を付与することにより、オブジェクトを他のユーザーと共有できます。

・4-ハッシュタグはCloudFront認証メカニズムではないため、CloudFrontを使用することはできません。

＜第33問＞
あなたのチームは、ログファイルに対して分析ジョブを毎日実行することを計画しているため、ログファイル用のストレージが必要です。ログのサイズと数は不明であり、データは24時間のみ保持されます。最も費用効果の高いストレージクラスを選択してください。

答え：3

1. S3 One Zone IA

2. S3 Intelligent Tiering

3. S3 標準

4. S3 Glacier Deep Archive


・1-One Zone IAストレージクラスには、オブジェクトごとの最小容量料金（128 KB）とGBごとの取得料金があります。

・2-Intelligent Tieringは使用に追加料金がかかり、短期間の要件ではメリットがない可能性があるため、正しくありません。

・3-S3標準が短期ストレージソリューションの最良の選択です。この場合、ログのサイズと数は不明であり、この段階でアクセスパターンを完全に評価することは困難です。したがって、S3標準を使用すると、費用効果が高く、すぐにアクセスでき、オブジェクトごとの取得料金や最小容量料金がないため最適です。

・4-Glacier Deep Archiveは検索料金がかかり、アーカイブからデータを取得するのに数時間かかるため不適切です。

＜第34問＞
あなたの会社は、AWSを使用して新しいWebベースのアプリケーションを開発しています。顧客にデータレコードへのアクセスを提供する新しい2層アプリケーションが展開されます。アプリケーションの応答性が高く、取得時間が最適化されていることが要件であり、その要件に適したパフォーマンスを提供できる永続的なデータストアを探しています。次のうち適切なものを選択してください。

答え：1

1. Redisエンジンを搭載したElastiCache

2. Kinesis Data Stream

3. Memcachedエンジンを搭載したElastiCache

4. マルチAZ構成のRDS


・1-ElastiCacheが提供するインメモリキャッシュを使用すると、読み取りが多い多くのアプリケーションワークロードや計算量の多いワークロードのレイテンシとスループットを大幅に向上させることができます。Redisは永続性を提供するため正しいです。

・2-Kinesis Data Streamsは、データのストリームを処理するために使用されます。永続的なデータストアではありません。

・3-Memcachedエンジンを搭載したElastiCacheは、永続性を提供していません。

・4-RDSは、ElastiCacheなどのインメモリデータストアにより適した取得時間を最適化する必要があるため、最適ではありません。

＜第42問＞
中小企業はオンプレミスでITシステムを運用してきましたが、運用効率を高めるためにAWSクラウドへの移行を計画しています。ソリューションアーキテクトとして、静的コンテンツと動的コンテンツの両方を備えたフラッグシップアプリケーション向けの費用対効果の高いサーバーレスソリューションを選択してください。

答え：4

1. RDSをデータベースとして使用して、EC2でWebアプリケーションの静的コンテンツと動的コンテンツの両方をホストし、CloudFrontは、地理的に分散したリージョンにコンテンツを配信するように設定する。

2. S3で静的コンテンツをホストし、動的コンテンツを生成するためにRDSでEC2を使用する。CloudFrontをEC2インスタンスの前に設定し、グローバルな配布に対応する。

3. S3でWebアプリケーションの静的コンテンツと動的コンテンツの両方をホストし、CloudFrontを使用して様々な国に配信する。

4. S3で静的コンテンツをホストし、動的コンテンツを処理するサーバレスWebアプリケーションにDynamoDBとLambdaを使用する。CloudFrontをLambdaの前に配置することで様々な地域へ配信できる。


・1-費用効果が高く最適なソリューションではありません。静的コンテンツはS3で費用対効果の高い方法で管理でき、EC2サーバーからコンテンツをフェッチする場合に比べてアクセスと配信が高速になるためです。

・2-EC2はサーバーレスサービスではありません。

・3-動的コンテンツのホストに適していないため、正しくありません。

・4-DynamoDBを備えたLambdaは、サーバーレスソリューションの正しい答えです。CloudFrontは、地理的に異なる場所に低レイテンシでコンテンツを配信することで、ユーザーエクスペリエンスの向上に役立ちます。S3は、Webアプリケーションの静的コンテンツを配布する費用対効果が高く高速な方法です。

＜第61問＞
企業は、ワークロードのライフサイクルを通じて大量の読み取りと書き込みを伴う分析ワークロードを実行します。同社の開発チームは、このワークロードをサポートするためのデータレイクとしてAmazonS3を使用することに関心を持っています。チームは、S3データ整合性モデルについてアドバイスするソリューションアーキテクトとしてあなたを雇いました。次のうち正しいものを選択してください。

答え：2

1. S3は、全てのGET、PUT、およびLIST操作で一貫性があり、最終的にはメタデータ情報を必要とする操作で一貫性がある。

2. S3は、全てのGET、PUT、およびLIST操作で一貫性がある。

3. S3は、全てのGET操作で一貫性があり、最終的にはPUTおよびLIST操作で一貫性がある。

4. S3は、全てのGETおよびPUT操作で一貫性があり、最終的にはLIST操作で一貫性を持つ。


・1-S3はGET、PUT、LIST操作、および変更オブジェクトタグ、ACL、またはメタデータの整合性が向上しました。したがって、このオプションは正しくありません。

・2-新しいオブジェクトの書き込みが成功した後、または既存のオブジェクトの上書きまたは削除後、後続の読み取りリクエストはすぐにオブジェクトの最新バージョンを受け取ります。S3はリスト操作にも強い一貫性を提供するため、書き込み後、変更を反映してバケット内のオブジェクトのリストをすぐに実行できます。

・3-S3はすべてのGET、PUT、およびLIST操作で一貫性が高いため、これら2つのオプションは正しくありません。

・4-S3はすべてのGET、PUT、およびLIST操作で一貫性が高いため、これら2つのオプションは正しくありません。

＜第62問＞
金融サービス会社は、S3バケットに機密データを保存します。コンプライアンスガイドラインでは、ファイルをサーバー側の暗号化で保存する必要があります。使用する暗号化はAES-256である必要があり、会社はS3暗号化キーを管理することを望んでいません。次のうち適切なものを選択してください。

答え：1

1. SSE-S3

2. SSE-KMS

3. SSE-C

4. クライアントサイド暗号化


・1-S3管理キー（SSE-S3）でサーバー側暗号化を使用して、各オブジェクトは強力な多要素暗号化を採用した一意のキーで暗号化されます。追加の保護手段として、定期的にローテーションするマスターキーを使用してキー自体を暗号化します。利用可能な最も強力なブロック暗号の1つであるAES-256を使用してデータを暗号化します。

・2-SSE-S3と同様であり、キーがいつ誰によって使用されたかの監査証跡も提供します。さらに、暗号化キーを自分で作成および管理するオプションがあります。

・3-クライアントが管理している暗号化キーを用いて、S3は、オブジェクトにアクセスするときにディスクに書き込むときの暗号化と復号化を管理します。

・4-クライアントサイドのデータを暗号化し、暗号化されたデータをAmazonS3にアップロードできます。この場合、暗号化プロセス、暗号化キー、および関連ツールを管理する必要があります。

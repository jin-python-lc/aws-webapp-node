試験1 - 分野別

エッジ・ルーティング

＜第2問＞
あなたの会社は画像配信アプリケーションを運用しています。画像配信を最適にするためにCloudFrontを利用していますが、コンテンツがエッジ側にない場合にどのような処理が発生するでしょうか。次の中で妥当なCloudFront処理を選択してください。

答え：2

1. コンテンツが保存された別のエッジへルーティングする。

2. CloudFrontがオリジンサーバへアクセスし、エッジにデータを取得する。

3. ユーザーに404エラーを返す

4. CloudFrontにリクエストをキャッシュし、データがエッジに来るまで待機する。


・1-CloudFrontはユーザーに近いエッジから配信します。そのため、配信先のユーザーに適したエッジに配信キャッシュがない場合はオリジンサーバーからデータを取得します。

・2-CloudFrontはユーザーから最も近いエッジロケーションにデータをキャッシュするコンテンツ配信ネットワーク（CDN）サービスであり、データ取得の待ち時間を短縮します。 データがエッジロケーションに存在しない場合は、データをオリジンサーバーから取得してから配信する可能性がありますが、次の配信以降はエッジロケーションにあるキャッシュを利用してデータが配信されます

・3-CloudFrontにデータがキャッシュされていなくてもユーザーに404エラーを返すことはありません。配信先のユーザーに適したエッジに配信キャッシュがない場合はオリジンサーバーからデータを取得します。

・4-CloudFrontはリクエストをキャッシュしてデータがエッジに来るまで待機することはありません。

＜第3問＞
AWS上に2つのEC2インスタンスを利用して構成したＷＥＢアプリケーションを構築しています。Route53を利用してランダムにそれぞれのサーバーにルーティングする設定を適応したいです。この要件を満たすルーティングポリシーを選定してください。

答え：2

1. 加重ルーティング

2. シンプル

3. レイテンシルーティング

4. フェイルオーバールーティング


・1-加重ルーティングは指定した比率で複数のリソースにトラフィックをルーティングする場合に使用します。単一のドメイン名 (example.com) またはサブドメイン名 (acme.example.com) に複数のリソースを関連付け、各リソースにルーティングされるトラフィックの数を選択できます。これは負荷分散や新しいバージョンのソフトウェアのテストなど、さまざまな目的に有用です。

・2-Route 53 コンソールでシンプルルーティングポリシーを選択した場合は、複数のレコードを同じ名前と型で作成することはできませんが、同一レコードに複数の値 (複数の IP アドレスなど) を指定することができます。そのため選択肢2が正解です。

・3-複数の AWS リージョンでアプリケーションがホストされている場合、ネットワークレイテンシーが最も低い AWS リージョンに基づいてリクエストを処理することで、ユーザーのパフォーマンスを向上させることができます。

・4-フェイルオーバーのルーティングにより、リソースが正常な場合にリソースにトラフィックをルーティングできます。また、最初のリソースが正常でない場合は別のリソースにルーティングできます。

＜第12問＞
Route53を使用してZone ApexレコードをELBと関連付ける必要があります。次のうち適切なレコードを選択してください。

答え：2

1. CNAMEレコード

2. ALIASレコード

3. NSレコード

4. MXレコード


・1-CNAMEレコードは、DNSクエリを任意のDNSレコードにリダイレクトできます。例えば、acme.example.com からzenith.example.com または acme.example.org にクエリをリダイレクトするCNAMEレコードを作成できます。

・2-ALIASレコードを使うことで、ホストゾーン (Zone Apex) と同じ名前のエイリアスレコードを作成できます。

・3-NSレコードは、ドメインのゾーン情報を管理するDNSサーバを定義するため、Zone Apexレコードを登録する場合には使用しません。

・4-MXレコードは、メールサーバーの名前を指定します。

＜第16問＞
あなたはAWSを利用したアプリケーション開発に携わっています。このアプリケーションを利用すると想定されるユーザーは世界中に分散しています。 そのため、ユーザーが地理的に近いリソースにルーティングされるようにする必要があります。この要件を満たすルーティングポリシーを選択してください。

答え：4

1. 複数値回答ルーティングポリシー

2. レイテンシールーティング

3. 位置情報ルーティング

4. 地理的近接性ルーティングポリシー


・1-複数値回答ルーティングポリシーはランダムに選択された最大8つのレコードを使用してDNSクエリに応答することができます。

・2-レイテンシールーティングポリシーはレイテンシーに基づいたトラフィックのルーティングする場合に使用します。

・3-位置情報ルーティングポリシーはコンテンツをそれぞれの位置にローカライズして、ユーザーに適した言語で表示します。DNSとIPアドレスの位置情報に基づいたルーティングのため、リソースの近さを考慮しません。

・4-地理的近接性ルーティングポリシーを利用することで、リソースの場所に基づいてトラフィックをルーティングし、必要に応じてトラフィックをリソースから別の場所のリソースに移動するようにできます。

＜第28問＞
多層Webアプリケーションのアプリケーション層は、現在、一つのEC2インスタンスで2つのWebサービスをホストしています。Webサービスはそれぞれ、異なるポートでトラフィックを受信します。リクエストパスに基づいてトラフィックをサービスにルーティングするために必要なサービスを選択してください。

答え：3

1. CloudFront

2. CLB

3. ALB

4. Route53


・1-CloudFrontは、コンテンツのキャッシュに使用されます。カスタムオリジンへのリクエストパスに基づいてルーティングできますが、設問ではコンテンツキャッシュについて言及していないため不正解です。

・2-CLBでホストベースまたはパスベースのルーティングを使用することはできません。

・3-Application Load Balancerは、レイヤー7（HTTP / HTTPS）プロトコルデータを使用して転送を決定できるElastic LoadBalancerの一種です。ALBは、パスベース（例：/indexなど）とホストベースのルーティング（例：example.com）の両方をサポートします。

・4-Amazon Route53はDNSサービスです。負荷分散に使用できますが、着信要求パスの情報に基づいてルーティングする機能はありません。

＜第56問＞
AWSは、グローバルインフラストラクチャの一部として多数のエッジロケーションを作成しました。次のうち、エッジロケーションを使用するメリットではないものを選択してください。

答え：4

1. エッジロケーションは、ファイルをアップロードする際のエンドユーザーエクスペリエンスを向上させるためにCloudFrontによって使用できる。

2. エッジロケーションは、CloudFrontがコンテンツを低レイテンシでグローバルに配信するために使用される。

3. エッジロケーションは、CloudFrontが最新の応答をキャッシュするために使用される。

4. CloudFrontは、エッジロケーションを使用してトラフィックを複数のインスタンスに分散し、レイテンシーを削減する。


・1-メリットとして正しいです。

・2-メリットとして正しいです。

・3-メリットとして正しいです。

・4-負荷を分散するために使用されるAWSサービスは、AWS Elastic Load Balancing（ELB）サービスです。

試験1 - 分野別

データベース

＜第1問＞
あなたは、1日あたり1000万人の顧客が利用する小売会社を経営しています。 顧客から得られるデータをビジネスに利用するため、高速にクエリを実行できるデータウェアハウスが必要です。 次のうち、要件に適したサービスを選択してください。

答え：1

1. Redshift

2. RDS

3. ElastiCache

4. Athena


・1-Redshiftは列指向アーキテクチャで、大量データの集計・分析に適したデータウェアハウスサービスです。

・2-RDSは汎用性の高いデータベースですが、大量データの分析には不向きです。

・3-ElastiCacheはインメモリデータベースのキャッシュサービスですが、設問においてキャッシュの利用では要件を満たせないため誤りです。

・4-Athenaは、標準SQLを使用してAmazonS3のデータを簡単に分析できるインタラクティブなクエリサービスです。データウェアハウスではありません。

＜第6問＞
業務アプリケーションを開発中です。このアプリケーションのデータベースには、多くの業務データを頻繁に作成・更新・削除できる性能が求められます。また、格納されたデータからレポートを作成するため、複数テーブルを結合してデータ取得する予定です。加えて、今後のデータ増加量に備えデータベースのストレージを自動拡張する機能も求められています。これらの要求を満たすサービスはどれですか。

答え：3

1. DynamoDB

2. S3

3. Aurora

4. Redshift


・1-DynamoDBは、プライマリキーを使用してテーブル内の各アイテムを一意に識別し、セカンダリインデックスを使用してクエリの柔軟性を高めます。noSQLのためSQLを利用することはできません。

・2-S3は、大容量のデータを格納できる耐久性の高いオブジェクトストレージサービスです。

・3-Auroraはマネージド型RDBです。頻繁にデータの更新・削除が行われるデータベースに向いており、SQLによるクエリも実行できます。また、Auroraはフルマネージドデータベースサービスのため、ストレージの自動拡張機能も備えています。

・4-Redshiftは列指向アーキテクチャで、大量データの集計・分析に適したデータウェアハウスサービスです。

＜第9問＞
EC2インスタンスとDynamoDBを利用してWebアプリケーションをホストし、ユーザーにサービスを提供しています。このサービスを利用しているユーザーから削除したはずのデータが表示されてしまうことがあると問い合わせを受けました。この問題を解決するために必要な対策を選択してください。

答え：4

1. DynamoDBストリームを利用する。

2. DynamoDBグローバルテーブルを利用する。

3. DynamoDB Accelerator (DAX)を利用する。

4. Consistent Readを有効化する。


・1-DynamoDBストリームは、DynamoDBのテーブルに対するデータ変更の履歴情報をイベントとして検出し、24時間保持する機能です。

・2-DynamoDBグローバルテーブルは、複数リージョンに配置されたデータベーステーブルでデータの整合性を保つ機能です。

・3-は、1秒あたり100万回単位のリクエストの応答時間を数ミリ秒で処理できるDynamoDB用インメモリキャッシュです。データを頻繁に参照する場合に使用することで高速な応答が可能になります。

・4-DynamoDBには結果整合性モデルという特性があります。そのため、変更されたデータは時間が経てば正しく反映されますが、タイミングによっては変更が反映されていない状態で読み込まれます。Consistent Readを有効化することにより読み取り一貫性を持たせることができます。

＜第19問＞
世界的に利用されているWEBサービスを運用しています。このアプリケーションの可用性を高めるため、最適なエンドポイントにトラフィックをルーティングするようにしたいです。この要件に最適なサービスを選択してください。

答え：1

1. Global Accelerator

2. Cloud Front

3. Route53

4. DynamoDB DAX


・1-AWS Global Acceleratorを使用しない場合、アプリケーションに到達するまでに多くのネットワークを経由しますが、使用することで2つのグローバルな静的パブリックIPが提供され、アプリケーションへの固定されたエントリポイントとして機能するため、アプリケーションへのパスを最適化できます。

・2-CloudFrontはグローバル配信CDNサービスです。エッジにキャッシュされたコンテンツを配信しますが、最適なエンドポイントへトラフィックルーティングするものではありません。

・3-Route53はDNSサービスです。ルーティングポリシーは様々ありますが、グローバルに最適なエンドポイントへトラフィックルーティングするものではありません。

・4-DynamoDB DAXはDynamoDBのキャッシュサービスです。最適なエンドポイントへトラフィックルーティングするものではありません。

＜第34問＞
あなたの会社は、AWSを使用して新しいWebベースのアプリケーションを開発しています。顧客にデータレコードへのアクセスを提供する新しい2層アプリケーションが展開されます。アプリケーションの応答性が高く、取得時間が最適化されていることが要件であり、その要件に適したパフォーマンスを提供できる永続的なデータストアを探しています。次のうち適切なものを選択してください。

答え：1

1. Redisエンジンを搭載したElastiCache

2. Kinesis Data Stream

3. Memcachedエンジンを搭載したElastiCache

4. マルチAZ構成のRDS


・1-ElastiCacheが提供するインメモリキャッシュを使用すると、読み取りが多い多くのアプリケーションワークロードや計算量の多いワークロードのレイテンシとスループットを大幅に向上させることができます。Redisは永続性を提供するため正しいです。

・2-Kinesis Data Streamsは、データのストリームを処理するために使用されます。永続的なデータストアではありません。

・3-Memcachedエンジンを搭載したElastiCacheは、永続性を提供していません。

・4-RDSは、ElastiCacheなどのインメモリデータストアにより適した取得時間を最適化する必要があるため、最適ではありません。

＜第40問＞
あなたの企業は、Application Load Balancerの背後にあるEC2インスタンスで実行される多層ソーシャルメディアアプリケーションを管理しています。インスタンスは、複数のアベイラビリティーゾーンにまたがるEC2 Auto Scalingグループで実行され、AmazonAuroraデータベースを使用します。ソリューションアーキテクトとして、リクエストレートの定期的な急上昇に対してアプリケーションをより回復力のあるものにすることが求められています。次のうち適切なソリューションを選択してください。

答え：2

1. AWS Sieldを利用する。

2. CloudFrontディストリビューションとAuroraレプリカを使用する。

3. Global Acceleratorを使用する。

4. Direct Connectを使用する。


・1-AWSShieldは、AWSで実行されているアプリケーションを保護するマネージド分散型サービス拒否（DDoS）保護サービスです。

・2-Auroraは、読み取り専用接続の負荷を、クラスター内にある数のAuroraレプリカに分散させることができます。Auroraレプリカは、可用性の向上にも役立ちます。最大15個のAuroraレプリカを、DBクラスターがAWSリージョン内にまたがるアベイラビリティーゾーン全体に分散できます。

・3-Acceleratorは、ローカルまたはグローバルユーザーのアプリケーションの可用性とパフォーマンスを向上させるサービスです。これは、アプリケーションロードバランサー、ネットワークロードバランサー、Amazon EC2インスタンスなど、単一または複数のAWSリージョン内のアプリケーションエンドポイントへの固定エントリポイントとして機能する静的IPアドレスを提供します。

・4-Direct Connectは、イントラネットとAmazonVPCの間で専用のプライベートネットワーク接続をするものです。

＜第45問＞
ある会社が最近、オンプレミスのデータセンターでデータベースの障害を経験しました。同社は現在、データの損失を最小限に抑え、すべてのトランザクションを少なくとも2つのノードに保存する、AWS上の信頼性の高いデータベースソリューションに移行したいと考えています。次のうちこの要件を満たすものを選択してください。

答え：2

1. RDS MySQL DBインスタンスを起動してから、別のアベイラビリティーゾーンにデータを同期的に複製するリードレプリカを作成する。

2. マルチAZ機能を有効化し、RDS MySQL DBインスタンスを起動し、データを同期的に複製する。

3. RDS MySQL DBインスタンスを起動してから、データを同期的に複製する別のリージョンにリードレプリカを作成する。

4. LambdaをトリガーしてデータをRDS MySQL DBインスタンスに同期的に複製するMySQL DBエンジンがインストールされたEC2インスタンスを起動する。


・1-データを同期的に複製するリードレプリカの作成について説明していますが、実際には、プライマリDBインスタンスに対して行われた更新はすべて非同期でリードレプリカにコピーされます。したがって正しくありません。

・2-RDSマルチAZ DBインスタンスをプロビジョニングすると、RDSは自動的にプライマリDBインスタンスを作成し、データを別のアベイラビリティーゾーンのスタンバイインスタンスに同期的に複製します。

・3-データを同期的に複製するリードレプリカの作成について説明していますが、実際には、プライマリDBインスタンスに対して行われた更新はすべて非同期でリードレプリカにコピーされます。したがって正しくありません。

・4-EC2インスタンスにデータベースをセットアップすることは、監視および管理する必要があるため信頼性に欠けます。

＜第46問＞
米国、ヨーロッパ、アジアに施設を持つグローバルな製造会社は、調達ワークフローを最適化するために新しい分散アプリケーションを設計しています。1つの大陸で予約された注文は、1秒以内にすべてのAWSリージョンに表示されます。データベースは、短い目標復旧時間（RTO）でフェイルオーバーを容易にできる必要があります。次のうち最も費用対効果の高いソリューションとして推奨すべきものを選択してください。

答え：4

1. クロスリージョンリードレプリカを使用してPostgreSQL用のRDSをプロビジョニングする。

2. DynamoDBグローバルテーブルをプロビジョニングする。

3. クロスリージョンレプリカを使用して、MySQL用のRDSをプロビジョニングする。

4. Auroraグローバルデータベースをプロビジョニングする。


・1-フェイルオーバーの場合、プロセスは自動ではないため、リードレプリカを手動でスタンドアロンデータベースインスタンスに起動する必要があります。したがって、RTOが非常に高くなるため適していません。

・2-DynamoDBグローバルテーブルは、クロスリージョンに対応した機能を提供しますが、SQLベースのデータベースに付属するデータアクセスの柔軟性の一部が失われます。DynamoDBグローバルテーブルのアクティブ/アクティブ構成により、アプリケーションがそのリージョンのテーブルに書き込み、データが複製されて他のリージョンのテーブルの同期が維持されるため、フェイルオーバーの概念はありません。DynamoDBグローバルテーブルは、特定の要件においてAuroraグローバルデータベースよりもはるかにコストのかかるソリューションです。

・3-フェイルオーバーの場合、プロセスは自動ではないため、リードレプリカを手動でスタンドアロンデータベースインスタンスに起動する必要があります。したがって、RTOが非常に高くなるため適していません。

・4-Auroraグローバルデータベースは、デフォルトのAuroraDBクラスターによって提供されるフェイルオーバーよりも包括的なフェイルオーバー機能を提供します。Auroraグローバルデータベースを使用することで、災害の計画と復旧をかなり迅速に行うことができます。(RTO:数分程度、RPO:秒単位)

＜第58問＞
企業は、AmazonRDSでデータベースを少なくとも3年間ホストする必要があります。次のうち、最も費用効果の高いものを選択してください。

答え：1

1. リザーブドインスタンス 一部前払い

2. リザーブドインスタンス　前払いなし

3. オンデマンドインスタンス

4. スポットインスタンス


・1-3年間ホストするということからリザーブドインスタンスが最適となります。一部前払いオプションを使用すると、リザーブド期間中、インスタンスの割引時間料金が請求されます。

・2-前払いなしオプションは、前払いを必要とせず、期間中の割引時間料金を提供しますが、前払いありと比べて時間料金が高くなります。

・3-オンデマンドは費用効果の高いソリューションではありません。

・4-スポットインスタンスはEC2のオプションです。RDSにはスポットオプションはありません。

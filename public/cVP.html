試験1 - 分野別

VPC

＜第11問＞
ある企業では、WebアプリケーションがホストされたEC2インスタンスをパブリックサブネット内に配置しています。そのアプリケーションは同じVPC内のプライベートサブネットのデータベース用EC2インスタンスにアクセスしています。ある時、データベースとして使用しているEC2インスタンスがパッチ処理を実行できていないことが判明しました。そのため、データベース用EC2のセキュリティを担保しながらDBミドルウェアのパッチ処理を可能にする必要があります。次のうち適切な構成を選択してください。

答え：3

1. プライベートサブネット内のEC2インスタンスのEBSボリュームタイプを、プロビジョンドIOPS SSDに変更する。

2. プライベートサブネットのルートテーブルにインターネットゲートウェイを追加する。

3. パブリックサブネット内にNATゲートウェイを配置し、プライベートサブネット内のEC2インスタンスと紐付ける。

4. プライベートサブネット内のEC2インスタンスをパブリックサブネット内に移行する。


・1-プロビジョンドIOPS SSDのEBSは、最大64,000IOPSに対応するI/O性能をもったボリュームタイプです。EBSボリュームタイプの変更は今回の設問と関係しません。

・2-プライベートサブネットのルートテーブルにインターネットゲートウェイを追加するとパブリックサブネット化してしまうので、セキュリティを担保できません。

・3-これにより、プライベートサブネットのインスタンスは、パブリックサブネットに存在するNATゲートウェイを使用して、インターネットにアクセスできます。データベースサーバーは、NAT ゲートウェイを通じてインターネットに接続してパッチ処理を行うことができますが、インターネットはデータベースサーバーへの接続を確立できません。

・4-プライベートサブネット内のEC2インスタンスをパブリックサブネット内に移行するとセキュリティが担保されないため不正解です。

＜第22問＞
アプリケーションを、2つのAZに分割されたプライベートサブネット上の2つのEC2インスタンス上で運用しています。このアプリケーションは、インターネット上で動作するCRM SaaSアプリケーションに接続する必要があります。SaaSアプリケーションのベンダーは、認証を登録されたIPアドレスのホワイトリストに制限しており、顧客ごとに2つのIPアドレスしか設定できません。SaaSアプリケーションへの認証を可能にするために、最も適切でコスト効率の良いソリューションは何でしょうか。

答え：4

1. 冗長化されたインターネットゲートウェイの設定と各サブネットのルーティングテーブルの更新。

2. Network Load Balancerを使用し、各AZに固定IPを設定する。

3. Elastic IPアドレスを持つ、インターネットに面した複数のアプリケーションロードバランサーを使用する。

4. Elastic IPアドレスを持つ各AZにNATゲートウェイを設定する。


・1-複数のインターネットゲートウェイを作成することはできないため誤りです。IGWはすでに冗長化されています。

・2-誤りです。ネットワークロードバランサーは、各AZに1つの固定IPアドレスを設定することができます（他のタイプのELBはできません）。しかし、接続はEC2インスタンスからSaaSアプリへのアウトバウンドで行われており、ELBはインバウンドの接続要求をEC2インスタンスに分配するために使用されているため、NLBを使用することは適切なソリューションではありません。

・3-誤りです。ALBは固定IPアドレスをサポートしておらず、プロキシ機能には適していません。

・4-NATゲートウェイは特定のAZで作成され、単一のElasticIPアドレスを関連付けることができます。

＜第30問＞
Amazon VPCに、複数のAmazonEC2インスタンスを起動しています。それらのインスタンスは、AmazonDynamoDBへのAPI呼び出しを行う必要があります。その際、API呼び出しがインターネットを通過しないようにできる構成を選択してください。

答え：1

1. DynamoDBのゲートウェイエンドポイントを作成する。

2. VPCとDynamoDBの間でVPCピアリング接続を設定する。

3. エンドポイントとして設定する新しいDynamoDBを作成する。

4. DynamoDBでグローバルテーブルを設定する。


・1-AmazonDynamoDBとAmazonS3は、インターフェースエンドポイントではなくゲートウェイエンドポイントをサポートします。ゲートウェイエンドポイントを使用して、VPCでエンドポイントを作成し、サービスへのアクセスを許可するポリシーをアタッチしてから、ルートテーブルを指定してでルートテーブルエントリを作成することで、インターネットを経由しない接続が可能となります。

・2-DynamoDBはリージョンサービスでありVPC内に無いため、VPCピアリング接続はできません。

・3-新しいDynamoDBテーブルを作成する必要がないため、正しくありません。

・4-DynamoDBグローバルテーブルは、複数リージョンに配置しているデータベーステーブルでデータの整合性を取るフルマネージド型のソリューションであり、設問とは関係ありません。

＜第41問＞
企業のエンジニアリングチームは、SQSを使用して基盤となるアプリケーションアーキテクチャのコンポーネントを分離したいと考えています。ただしチームは、パブリックインターネットを介してSQSにアクセスするVPCにバインドされたコンポーネントをセキュアなものにしたいと考えています。ソリューションアーキテクトとして、このケースに対処するために選択するべきソリューションは次のうちどれですか？

答え：2

1. VPN接続を使用してSQSにアクセスする。

2. VPCエンドポイントを使用してSQSにアクセスする。

3. インターネットゲートウェイを使用してSQSにアクセスする。

4. NATインスタンスを使用してSQSにアクセスする。


・1-既存のインフラストラクチャはAWSクラウド内にあるため、VPN接続は必要ありません。

・2-SQSのVPCエンドポイントは、AWS PrivateLinkを利用しています。これは、サポートされているAWSサービスにVPCをプライベートに接続できる高可用性でスケーラブルなテクノロジーです。

・3-チームはパブリックインターネットを使用してAmazonSQSにアクセスすることを望まないため、誤りです。

・4-NATインスタンスは、プライベートサブネット内のインスタンスへのインターネットアクセスを提供するために使用されるため、誤りです。

＜第43問＞
メディア企業はロサンゼルスに本社を置き、VPCへのDirectConnect接続をしているオンプレミスのデータセンターを持っています。サンフランシスコとマイアミのブランチオフィスは、サイト間VPN接続を使用してVPCに接続します。同社は、支店間および本社との間でデータを送受信するためのソリューションを探しています。ソリューションアーキテクトとして、このケースに対処するために提案すべきAWSサービスを選択してください。

答え：2

1. VPCエンドポイントを使用してSQSにアクセスする。

2. VPN CloudHub

3. VPCピアリング

4. ソフトウェアVPN


・1-VPCエンドポイントを使用すると、インターネットゲートウェイ、NATデバイス、VPN接続、またはAWS Direct Connect接続を必要とせずに、サポートされているAWSサービスおよびAWSPrivateLinkを利用したVPCエンドポイントサービスにVPCをプライベートに接続できます。VPCエンドポイントを使用する場合、VPCと他のAWSサービス間のトラフィックはAmazonネットワークを離れないため、会社のリモートブランチオフィス間でデータを送受信することはできません。

・2-複数のAWSサイト間VPN接続がある場合は、VPNCloudHubを使用してサイト間の安全な通信を提供できます。これにより、リモートサイトはVPCだけでなく、相互に通信できるようになります。

・3-VPCピアリング接続は、プライベートIPv4アドレスまたはIPv6アドレスを使用して2つのVPC間でトラフィックをルーティングできるようにする2つのVPC間のネットワーク接続です。AWSネットワーク内の2つのVPC間の接続を容易にするため、会社のリモートブランチオフィス間でデータを送受信することはできません。

・4-SoftwareVPNはリモートネットワークとAmazonVPC間の接続を処理するだけなので、会社のリモートブランチオフィス間でデータを送受信するために使用することはできません。

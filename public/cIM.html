＜第13問＞
IAMユーザ新規作成時のデフォルト権限について正しいものを選択してください。

答え：2

1. 全ての権限が付与されているため、権限を変更する必要がある。

2. 何の権限も付与されていない。

3. IAMの閲覧権限のみ付与される。

4. CloudWatchの閲覧権限のみ付与される。


・1-IAMユーザ新規作成時のデフォルト権限では、何の権限も付与されていません。

・2-IAMユーザ新規作成時のデフォルト権限では、何の権限も付与されていません。

・3-IAMユーザ新規作成時のデフォルト権限では、何の権限も付与されていません。

・4-IAMユーザ新規作成時のデフォルト権限では、何の権限も付与されていません。

＜第23問＞
EC2ローンチタイプを使用するECSコンテナインスタンス上で実行されるアプリケーションは、Amazon DynamoDBにデータを書き込むためのパーミッションを必要とします。アプリケーションを実行している特定のECSタスクにのみ、これらのパーミッションを割り当てるにはどうすればよいでしょうか。

答え：2

1. セキュリティーグループを使用してDynamoDBへのアウトバウンド接続を許可し、それをコンテナインスタンスに割り当てる。

2. DynamoDBへのアクセス許可を持つIAMポリシーを作成し、taskRoleArnパラメーターを使用してタスクに割り当てる。

3. DynamoDBへのアクセス許可を持つIAMポリシーを作成し、コンテナインスタンスにアタッチする。

4. ECSTaskExecutionRolePolicyを変更して、DynamoDBのパーミッションを追加する。


・1-DynamoDBへのアウトバウンド接続を許可するにはセキュリティグループが必要ですが、問題はDynamoDBにデータを書き込むためのアクセス許可を割り当てる方法を尋ねることであり、セキュリティグループはそれらのアクセス許可を提供できません。

・2-Amazon ECSで特定のタスクのパーミッションを指定するには、IAM Roles for Tasksを使用する必要があります。

・3-この場合権限は、インスタンスで実行されているすべてのタスクとインスタンス自体に適用されます。

・4-これは、コンテナエージェントがコンテナイメージのプル、ログファイルの書き込みなどを行うために使用します。

＜第44問＞
小売企業は、AWS上でITインフラストラクチャを管理しています。同社は、AWSアカウントを発行し、EC2インスタンスやRDSデータベースなどのリソースを使用する複数の部門を管理するためにAWS管理組織を設立しました。同社は、集中管理されたVPCをすべての部門に提供したいと考えています。次のうち適切なものを選択してください。

答え：3

1. VPCピアリングを使用して、AWS Organizationsの同じ親組織に属する他のAWSアカウントとVPCを共有する。

2. VPCピアリングを使用して、AWS Organizationsの同じ親組織に属する他のAWSアカウントと１つ以上のサブネットを共有する。

3. VPC共有を使用して、AWS Organizationsの同じ親組織に属する他のAWSアカウントと1つ以上のサブネットを共有する。

4. VPC共有を使用して、AWS Organizationsの同じ親組織に属する他のAWSアカウントとVPCを共有する。


・1-VPCピアリングは、集中管理されたVPCを提供できません。さらに、AWSオーナーアカウントはVPC自体を別のAWSアカウントと共有することはできません。したがって、正しくありません。

・2-VPCピアリングは、集中管理されたVPCを提供できません。したがって、正しくありません。

・3-VPC共有を使用すると、複数のAWSアカウントでEC2インスタンス、RDSデータベース、Redshiftクラスター、Lambda関数などのアプリケーションリソースを共有および集中管理されたAmazon Virtual Private Cloud（VPC）に作成できます。

・4-VPCを所有するアカウントは、同じ組織に属する他のアカウントと1つ以上のサブネットを共有しますが、所有者アカウントはVPC自体を共有できません。したがって、正しくありません。

＜第49問＞
あなたの組織には、AWSクラウドインフラストラクチャを運用する多数のエンジニアがいます。あなたは、チームを編成し、各チームに適切なアクセス許可を割り当てたいと考えています。その際利用すべきサービスを選択してください。

答え：4

1. IAM ユーザー

2. AWS Organizations

3. IAM ロール

4. IAM グループ


・1-IAMユーザーは、AWS Management Consoleにアクセスするために、ルートユーザーから払い出されるユーザーアカウントです。

・2-AWS Organizationsは、IAMユーザー（従業員）ではなく、AWSアカウントをグループ化するために使用します。

・3-AMロールは、特定のアクセス許可を持つアカウントで作成できるIAMIDです。

・4-IAMグループは、1つの単位として管理されるIAMユーザーのコレクションです。グループを使用すると、複数のユーザーのアクセス許可を管理できます。これにより、それらのユーザーのアクセス許可を簡単に管理できます。

＜第53問＞
AWS共有責任モデルに関して正しいものを選択してください。

答え：2

1. IaasサービスのセキュリティはAWSの責任である。

2. 責任範囲は使用するサービスによって異なる。

3. ゲストOSへのパッチ適用は常にAWSの責任である。

4. マネージドサービスのセキュリティはユーザーの責任である。


・1-サービスとしてのインフラストラクチャ（IaaS）であるAWS製品は完全にユーザーの管理下にあり、必要なすべてのセキュリティ構成および管理タスクはユーザーが実行する必要があります。

・2-選択したAWSサービスによって責任が異なる場合があります。

・3-ゲストOSへのパッチ適用は、マネージドサービス（Amazon RDSなど）に対してのみAWSが責任を負います。ユーザーは、他のサービス（Amazon EC2など）のゲストOSにパッチを適用する責任があります。AWSは、基盤となるホストにパッチを適用し、ファームウェアをアップグレードし、AmazonEC2を含むすべてのサービスのインフラストラクチャ内の欠陥を修正する責任があります。

・4-AWSは、マネージドサービスのセキュリティ構成を担当します。
